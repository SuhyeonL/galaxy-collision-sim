# step 1 데이터셋 만들기
python으로 데이터 생성 -> 파일 저장 후 c++, cuda에서 사용
두 은하의 입자들 초기 상태
각 입자는 다음 데이터를 가짐
- 위치(x, y, z)
- 속도(vx, vy, vz)
- 질량(mass)

가우시안 분포로 위치 생성
np.random.normal() 함수 사용
np.random.normal(loc, scale, size)
loc -> 평균 (은하 중심 좌표)
scale -> 표준편차 (은하 크기, 클수록 넓게 퍼짐)
size -> 생성할 입자 수 N
loc 근처의 숫자들이 가우시안 분포로 N개의 데이터 생성

y축만 좁게 눌러서 원반 모양
두 은하 위치를 서로 반대편에 배치
각 은하에 반대 방향 속도 부여
.bin 파일로 저장

## 은하 데이터 셋
### 위치
- loc = 은하 중심 위치
- 두 은하가 나중에 충돌해야 하니까 서로 반대편에 배치
scale = 은하 크기 (퍼짐 정도)
- 너무 작으면 입자들이 한 점에 뭉쳐있음
- 너무 크면 입자들이 너무 흩어져서 은하처럼 안 보임
- 두 은하의 scale이 같아야 충돌이 자연스러움
- 위치는 은하 모양을 만들어야 해서 가우시안 분포가 필요함
- 가우시안 분포 중심에 몰리고 바깥으로 갈수록 적어져야 은하 모양 생성
- 속도와 질량은 필요 없음
### 속도
- 고정값 
- 두 은하가 서로 접근하는 방향으로 일정한 값을 주면 됨
### 질량
- 고정값 or 균일 분포

## 데이터 셋 시각화
나선 모양은 N-body 시뮬레이션 돌리면 중력 상호작용으로 입자들이 움직이면서 자연스럽게 나타남
데이터 생성 단계에서는 타원형으로 나타남

## 단순 데이터 사용
현재 프로젝트는 물리적 정확도가 아닌 N-Body simulation에서 cuda 성능 최적화임
일단 가장 단순한 x축에서만 일정하게 움직이는 은하로 가정 -> x축에서만 움직이게 설정, 속도, 질량 일정
단순한 설정으로 시작하고 여유가 생길 시 다음 추가
- 은하마다 질량 다르게
- 회전 속도 추가
- 약간 기울어진 각도로 충돌
